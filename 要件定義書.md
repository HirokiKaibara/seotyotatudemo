# 要件定義書

## 1. プロジェクト概要

本プロジェクトは、発注担当者が外注加工見積の価格検討を行うための業務用デモ画面を作成するものである。

対象業務は、図面PDFを外注業者へ提示して見積を取得する加工案件、過去購買管理データを参照した実績単価確認、SUS材の原料価格・為替変動を踏まえた価格確認である。

現時点のデモでは、AI連携、PDF解析、DB保存、外部API取得、スクレイピングは行わない。ローカルのサンプルデータと簡易スコアにより、将来的な業務イメージを一画面で確認できるようにする。

本開発では、図面PDFをDBに保存し、蓄積した図面情報や過去購買管理データから、AIが必要な情報や類似加工事例を引き出せる構成を目指す。

## 2. 機能要件

### 2.1 見積内容入力

- 見積注番、案件名、品名、寸法、材質、数量、業者名、その他キーワードを入力できる。
- 材質は「SUS304」「SUS316L」から選択できる。
- 図面PDFをファイル選択できる。
- 現デモでは、PDFはアップロード・解析せず、選択ファイル名のみ表示する。

### 2.2 現行見積作成画面イメージ表示

- 見積内容入力の上部に、現行の見積作成画面を模したレイアウトと表を表示する。
- 既存業務の流れを画面上部に残すことで、教育しやすさと操作イメージのしやすさを高める。
- 現行見積作成画面イメージはデモ表示のみとし、入力や保存処理は行わない。

### 2.3 調達参考情報表示

- 「調達参考情報を表示」ボタンで、入力内容に基づく参考情報を表示する。
- 入力条件を変更した場合は、再度表示操作を行う。

### 2.4 実績表表示

- 過去購買管理データのサンプルから、今回の見積内容に近い実績を表示する。
- 選別条件は、品名、寸法、材質、数量、業者名、その他キーワードとする。
- 現デモではAI連携は行わず、簡易スコアで近い実績を選別する。
- 1見積注番につき最大3件の引用実績を表示する。
- 実績表には、見積注番、引用注番、品名、寸法、材質、業者名、数量、実績単価、一致項目、スコアを表示する。

### 2.5 原料・為替情報表示

- ニッケル価格、クロム価格、モリブデン価格、為替を表示する。
- 現デモではネット取得は行わず、サンプルデータを表示する。
- 将来的には、許可済みURL、API、RSS、CSVなどから取得する。

### 2.6 SUS材単価表表示

- SUS304、SUS316Lの単価表を表示する。
- 現デモでは直近3日分のみ表示する。
- 前日比と傾向を表示する。
- 将来的には、原料価格と為替の変動を毎日読み取り、SUS材単価表を自動作成する。

### 2.7 比較結果表示

- 実績表に表示された過去実績の平均単価を算出する。
- 選択された材質のSUS材参考単価を表示する。
- 実績表の平均単価とSUS材参考単価の差額を表示する。
- 価格検討メモを表示する。

## 3. 非機能要件

- ローカル環境で動作することを優先する。
- Webアプリ形式で提供する。
- 一画面で主要情報を確認できる構成とする。
- デザインはシンプルな業務システム風とする。
- AI感を強く出さず、業務担当者が確認するための参考情報として表現する。
- DB、認証、Firebase、外部APIは使用しない。
- 実際のスクレイピング、PDF解析、AI連携は行わない。
- 各自作ファイルは原則400行以内に保つ。

## 5. 技術選定

### 5.1 フロントエンド

- Next.js 16.2.5
- React 19.2.6
- TypeScript 6.0.3
- CSS

### 5.2 データ管理

- 現デモではTypeScriptファイル内のサンプルデータを使用する。
- 過去購買管理データは `sample/purchase-records.ts` に定義する。
- 原料・為替データ、SUS材日次単価データは `sample/stainless-market.ts` に定義する。

### 5.3 ディレクトリ方針

- アプリ本体は `src` 配下に配置する。
- 画面機能は `src/features/procurement` 配下に分割する。
- 共通型は `src/types` 配下に配置する。
- デモ用サンプルデータはルート直下の `sample` に配置する。

## 6. 開発・運用体制

### 6.1 開発方針

- 最低要件を満たすデモとして実装する。
- ローカルで動作することを優先する。
- 将来的なAI連携、PDF保存、外部データ取得に差し替えやすい構成にする。
- 画面、計算処理、サンプルデータ、型定義を分離する。

### 6.2 確認コマンド

```powershell
npm run dev
npm run typecheck
npm run build
```

### 6.3 運用前提

- 現デモは本番運用を想定しない。
- データ更新はコード内サンプルの変更で行う。
- 認証や権限管理は行わない。

## 7. 今後の課題、リスク

### 7.1 今後の課題

- 図面PDFをDBに保存し、図面情報を蓄積する仕組みの設計。
- 蓄積済み図面情報から類似加工事例を検索する仕組みの設計。
- AI連携を行う場合の入力データ、出力形式、根拠表示の設計。
- 購買管理データとの連携方式の検討。
- ニッケル、クロム、モリブデン、為替の取得元選定。
- SUS304、SUS316L単価表の算出ロジック定義。

### 7.2 リスク

- 図面PDFの内容抽出は現時点では予定していないが、将来的に実装する場合は抽出精度が実務利用に影響する可能性がある。
- 類似案件の選別理由が不明確だと、発注担当者が価格査定に利用しづらい。
- 原料価格や為替の取得元が不安定な場合、単価表の信頼性が下がる。
- 外部データ取得には、利用規約、契約、取得頻度、保存可否の確認が必要になる。
- AIが価格を断定しているように見える表現は、業務上の誤解につながる。
- 本番化時には、DB設計、認証、権限管理、監査ログ、バックアップが必要になる。
