# 要件定義書

## 1. プロジェクト概要

本プロジェクトは、発注担当者が外注加工見積の価格検討を行うための、ローカル動作の業務用デモ画面を作成するものである。

主な改善対象は、インターネット上のニッケル、クロム、モリブデン、為替レートなどの相場情報を手作業で確認し、Excelへ転記している作業である。本デモでは、その確認作業を一画面上の参考情報表示として再現し、将来的な業務改善イメージを確認できるようにする。

現時点のデモでは、ニッケル、クロム、モリブデン、為替レートをネット取得し、取得できない場合はローカルのモックデータで表示する。SUS材単価表と比較結果の参考単価は、ローカルのサンプルデータで表示する。DB、AI連携、RSS取得は行わない。その他素材の相場情報、過去購買実績、既存見積明細もローカルのサンプルデータで表示する。

## 2. 現デモの位置付け

- ローカルPC上で動作する一画面の簡易デモとする。
- 製品版ではネット接続を前提にするが、デモ版ではネット未接続でも画面表示と操作確認を継続できるようにする。
- 価格を確定するシステムではなく、発注担当者が確認する参考情報をまとめて表示する画面とする。
- 現行の見積作成画面に近い見た目を残し、既存業務からの変更点を説明しやすくする。

## 3. 機能要件

### 3.1 現行見積作成画面イメージ表示

- 画面上部に、現行の見積作成画面を模したレイアウトと明細表を表示する。
- 見積依頼先コード、出力区分、国内外区分、依頼条件、検査区分を表示する。
- 明細表には、注文番号、オーダーNo、工程名、品名、寸法、材質、数量、価格、要求納期、納入先などを表示する。
- 明細表は横スクロールに対応し、列幅調整ができる。
- 工程名、品名、材質は画面上で変更できる。
- データ引継チェックにより、対象行の選択イメージを確認できる。

### 3.2 反映操作

- 「反映」ボタンを押すと、下部の参考情報エリアを表示する。
- 反映前は、下部エリアに案内文を表示する。
- 反映後は、現行見積明細、相場情報、SUS材単価表、過去購買実績をもとにした参考情報を表示する。

### 3.3 原料・為替情報表示

- ニッケル価格、クロム価格、モリブデン価格、為替レートを表示する。
- 見積明細に含まれる材質に応じて、鉄鉱石、原料炭、鉄スクラップ、フェロマンガン、フェロシリコンなどの関連相場も表示する。
- ニッケル、モリブデンはDailyMetalPriceの日次価格、クロムはScrapMonsterの公開価格を取得し、USD/JPYで円換算して表示する。
- 為替レートは公開為替APIからUSD/JPYを取得して表示する。
- ネット不通、取得元エラー、タイムアウトの場合は、該当項目をローカルのモックデータで表示する。
- その他素材の関連相場は、見せ方確認用として常にモックデータを表示する。
- オンライン時は参照元名を外部リンクとして表示する。
- ネット未接続時は参照元名を表示し、外部リンク遷移は行わない。

### 3.4 SUS材単価表表示

- SUS304、SUS316Lの直近3日分の単価表を表示する。
- 前日比と傾向を表示する。
- `sample/stainless-market.ts` のサンプル単価表を使用し、日付は表示日の前日から3日分に差し替えて表示する。
- ネット取得した相場値はSUS材単価表へ反映しない。

### 3.5 実績表表示

- 過去購買管理データのサンプルから、今回の条件に近い実績を最大3件表示する。
- 選別条件は、品名、寸法、材質、数量、業者名、その他キーワードとする。
- 現デモでは、簡易スコアにより近い実績を選別する。
- 実績表には、見積注番、引用注番、品名、寸法、材質、業者名、数量、実績単価、一致項目、スコアを表示する。

### 3.6 比較結果表示

- 表示された過去実績の平均単価を算出する。
- 選択材質に応じたSUS材参考単価を表示する。
- 実績表の平均単価とSUS材参考単価の差額を表示する。
- 前日比を表示する。
- 価格検討メモを表示する。

### 3.7 オフラインデモ対応

- アプリ本体、画面、サンプルデータはローカルに保持する。
- 事前にビルド済みであれば、ネット未接続でも `npm run start` で画面確認できる。
- ブラウザのオンライン状態、または `?offline=1` のURLパラメータにより、未接続時の動作確認ができる。
- 未接続時でも、画面上に不要な注意文を出さず、製品版想定に近い見た目を維持する。

## 4. 非機能要件

- ローカル環境で動作することを優先する。
- Webアプリ形式で提供する。
- 一画面で主要情報を確認できる構成とする。
- デザインはシンプルな業務システム風とする。
- DB、認証、Firebase、RSS取得は使用しない。
- 外部データ取得は、ニッケル、クロム、モリブデン、為替レートの表示に限定する。
- 実際のPDF解析、ファイル保存、AI連携は行わない。
- 各自作ファイルは原則400行以内に保つ。

## 6. 技術選定

### 6.1 フロントエンド

- Next.js 16.2.5
- React 19.2.6
- TypeScript 6.0.3
- CSS

### 6.2 データ管理

- 現デモではTypeScriptファイル内のサンプルデータを使用する。
- 既存見積明細データは `sample/existing-estimate-rows.ts` に定義する。
- 過去購買管理データは `sample/purchase-records.ts` に定義する。
- ニッケル、クロム、モリブデン、為替のモックデータとSUS材参考単価のフォールバックデータは `sample/stainless-market.ts` に定義する。
- 材質別の関連相場データは `sample/material-market-indicators.ts` に定義する。
- ネット取得処理は `src/app/api/market-indicators/route.ts` に定義する。

### 6.3 ディレクトリ方針

- アプリ本体は `src` 配下に配置する。
- 画面機能は `src/features/procurement` 配下に分割する。
- 共通型は `src/types` 配下に配置する。
- デモ用サンプルデータはルート直下の `sample` に配置する。
- 資料はルート直下または `docs` 配下に配置する。

## 7. 開発・確認方法

### 7.1 確認コマンド

```powershell
npm run typecheck
npm run build
npm run start
```

### 7.2 起動URL

```text
http://localhost:3000/
```

### 7.3 オフライン表示確認

```text
http://localhost:3000/?offline=1
```

実際にネット接続を切って確認する場合は、事前に `npm run build` を実行してから `npm run start` で起動する。

### 7.4 ネット不通時の実行手順

本デモはビルド時に外部データ取得を行わないため、依存パッケージがローカルに入っていれば、ネット不通時でも型チェック、ビルド、起動確認ができる。

ネット不通時に実行できる条件は以下のとおりである。

| 条件 | 内容 |
| --- | --- |
| `node_modules` が存在すること | 事前にネット接続ありの状態で `npm install` が完了している必要がある |
| ビルド済みで起動する場合は `.next` が存在すること | `npm run start` は、事前に `npm run build` が完了している必要がある |
| 3000番ポートが空いていること | 他の開発サーバーが起動中の場合は停止してから実行する |

ネット不通時の確認手順は以下とする。

```powershell
npm run typecheck
npm run build
npm run start
```

各コマンドの扱いは以下のとおりである。

| コマンド | ネット不通時の実行可否 | 補足 |
| --- | --- | --- |
| `npm run typecheck` | 実行可能 | ローカルのTypeScriptとソースコードを確認する |
| `npm run build` | 実行可能 | ビルド時は外部データ取得を行わないため、ローカルデータのみでビルドできる |
| `npm run start` | 実行可能 | 事前に `npm run build` が完了している必要がある |

起動後は以下のURLをブラウザで確認する。

```text
http://localhost:3000/
```

未接続時の表示を強制的に確認する場合は、以下のURLを使用する。

```text
http://localhost:3000/?offline=1
```

なお、`node_modules` が存在しない状態では、ネット不通時に `npm install` ができないため、事前準備としてネット接続ありの状態で依存パッケージをインストールしておく必要がある。

## 8. 運用前提

- 現デモは本番運用を想定しない。
- データ更新はコード内サンプルの変更で行う。
- 認証や権限管理は行わない。
- ニッケル、クロム、モリブデン、為替レートはネット取得を行い、取得できない場合はモックデータで表示する。
- その他素材の関連相場は、表示確認用のモックデータとして扱う。

## 9. 今後の課題、リスク

### 9.1 今後の課題

- 購買管理データとの連携方式を検討する。
- ニッケル、クロム、モリブデン、為替の取得元を選定する。
- 取得元の利用条件、取得頻度、保存可否を確認する。
- SUS304、SUS316L単価表の正式な算出ロジックを定義する。
- 図面PDFや見積依頼書を扱う場合の保存、参照、権限管理を検討する。
- 本番運用時の認証、権限、監査ログ、バックアップを設計する。

### 9.2 リスク

- 原料価格や為替の取得元が不安定な場合、参考単価の信頼性が下がる。
- 相場情報の更新タイミングと購買判断のタイミングがずれる可能性がある。
- 過去実績の選別理由が不明確だと、発注担当者が価格検討に利用しづらい。
- 外部データ取得には、利用規約、契約、取得頻度、保存可否の確認が必要になる。
- 参考単価が確定価格のように受け取られると、業務上の誤解につながる。
