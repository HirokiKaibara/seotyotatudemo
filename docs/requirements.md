# 製品調達支援デモ 要件定義

## 1. 概要

発注担当者が、図面PDFを伴う外注加工見積に対して、過去外注加工事例、購買管理データ由来の実績表、SUS材の日次単価トレンドを一画面で確認できるデモ画面を作成する。

本デモは最低要件の画面確認を目的とする。AI連携、PDF解析、外部API、スクレイピング、DB保存は行わず、ローカルのサンプルデータと簡易スコアで動作を表現する。

## 2. 本開発で目指す姿

### 2.1 図面PDFデータの蓄積

- 設計図面をPDF等のファイルに落とし込み、外注業者へ見積依頼するケースを対象にする。
- 本開発では、図面PDFデータをDBに保存し、図面情報を蓄積する。
- 必要に応じてAI連携を行い、AIがDBから必要な図面情報や類似加工事例を引き出せるようにする。
- 別案件で図面内容をもとに見積を取る際、蓄積済みPDFデータから類似加工事例のオーダを自動選別する。
- AIが候補として上げた過去外注加工事例は、発注担当者が精度を評価し、価格査定に役立てる。

### 2.2 見積依頼書と別紙実績表

- 見積依頼書が出力される際、業者向け見積依頼書とは別に実績表を自動出力する。
- 実績表には、各見積注番ごとに過去の実績単価を3件程度表示する。
- 実績表には、引用元となった過去注番を表示する。
- 実績表は、過去の購買管理データから品名、寸法、材質、その他キーワードをもとに近い実績を自動選別する。
- 引合先から見積回答があった後、発注担当者は実績表を参考に価格を決め、価格交渉に利用する。

### 2.3 SUS材単価表の自動作成

- ネットからニッケル価格、クロム価格、モリブデン価格、為替を取得する。
- それらの変動を毎日読み取り、SUS304、SUS316Lの単価表を自動作成する。
- 日替のSUS材価格をトレンドとして確認できるようにする。

## 3. 現デモの範囲

- 図面PDFはファイル選択とファイル名表示のみ行う。
- PDF内容の解析、保存、AI連携は行わない。
- 本ページでは、見積依頼書の出力UIやPDF蓄積状態の専用表示エリアは扱わない。
- 見積内容入力の上部に、現行の見積作成画面を模したレイアウトと表を表示する。
- 類似加工事例の選別は、品名、寸法、材質、数量、業者名、その他キーワードによる簡易スコアで代替する。
- 実績表は、見積注番ごとに最大3件の引用実績を表示する。
- ニッケル価格、クロム価格、モリブデン価格、為替、直近3日分のSUS材単価表はサンプルデータから表示する。

## 4. 画面構成

画面タイトルは「製品調達支援デモ」とする。

1. 現行見積作成画面イメージエリア
   - 見積依頼先
   - 検索条件
   - 依頼条件
   - 検査区分
   - 見積明細表

2. 見積内容入力エリア
   - 見積注番
   - 案件名
   - 品名
   - 寸法
   - 材質
   - 数量
   - 業者名
   - その他キーワード
   - 図面PDF選択

3. 原料・為替情報エリア
   - ニッケル価格
   - クロム価格
   - モリブデン価格
   - 為替
   - 前回比
   - 取得日

4. SUS材単価表エリア
   - 日付
- SUS304単価
   - SUS304前日比
   - SUS316L単価
   - SUS316L前日比
   - 傾向

5. 実績表エリア
   - 見積注番
   - 引用注番
   - 品名
   - 寸法
   - 材質
   - 業者名
   - 数量
   - 実績単価
   - 一致項目
   - スコア

6. 比較結果エリア
   - 実績表の平均単価
   - SUS材参考単価
   - 前日比
   - 差額
   - 価格検討メモ

## 5. 機能要件

### 5.1 見積内容入力

- 見積注番、案件名、品名、寸法、材質、数量、業者名、その他キーワードを入力できる。
- 材質は「SUS304」「SUS316L」から選択する。
- 図面PDFはファイル選択のみ行い、選択したファイル名を表示する。

### 5.2 実績表

- 「調達参考情報を表示」ボタンを押すと、入力条件に近い過去実績を表示する。
- 1見積注番につき最大3件の引用実績を表示する。
- 引用元の過去注番を表示する。

### 5.3 SUS材単価表

- SUS304、SUS316Lの日次単価表を表示する。
- 現デモでは直近3日分のみ表示する。
- 前日比と傾向を表示する。
- 現時点ではサンプルデータを利用する。

## 6. 非機能要件

- ローカル環境で動作することを優先する。
- 一画面で完結するWebアプリ形式とする。
- デザインはシンプルな業務システム風とする。
- 各自作ファイルは原則400行以内に保つ。
- DB、認証、Firebase、外部APIは使用しない。
- 実際のスクレイピング、PDF解析、AI連携は行わない。

## 7. 将来拡張

- 図面PDFをDBに保存し、図面情報を蓄積できるようにする。
- DBに蓄積した図面情報から、AIが必要な情報や類似加工事例を引き出せるようにする。
- 必要に応じて、見積依頼書や別紙実績表のファイル出力を別画面または別機能として実装する。
- 過去購買管理データを基幹システム、CSV、APIから取得できるようにする。
- 原料価格と為替を許可済みURL、API、RSS、CSV取得に差し替える。
- SUS304、SUS316Lの日次単価表を自動生成し、履歴として蓄積できるようにする。
